PINN con Condiciones de Frontera Variables

Extensión de tesis: Solución de la ecuación de calor unidimensional estacionaria empleando una red neuronal físicamente informada

📖 Descripción del Proyecto

Partiendo de mi tesis de licenciatura "Solución de la ecuación de calor unidimensional estacionaria empleando una red neuronal físicamente informada", desarrollé una extensión que permite resolver la misma ecuación pero con condiciones de frontera variables en un rango determinado.

Las Physics-Informed Neural Networks (PINNs) son redes neuronales que integran leyes físicas en su entrenamiento. En lugar de aprender únicamente a partir de datos (como una red convencional), las PINN incorporan la ecuación diferencial que gobierna el fenómeno como parte de la función de pérdida. Esto permite entrenar sin datos de solución, garantizar la consistencia física del problema y generalizar a nuevas condiciones de frontera sin reentrenar la red.

🔬 Metodología

Arquitectura de la Red

La arquitectura PINN incorpora la ecuación de calor como término de pérdida en la función de costo. Implementé una capa de entrada que parametriza las condiciones de frontera, permitiendo que la red generalice a múltiples escenarios sin necesidad de reentrenamiento completo.

class HeatPINN(nn.Module):
    def __init__(self):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(1, 50), nn.Tanh(),
            nn.Linear(50, 50), nn.Tanh(),
            nn.Linear(50, 50), nn.Tanh(),
            nn.Linear(50, 1)
        )
    
    def forward(self, x, T_left, T_right):
        D = x * (1 - x)
        N = self.net(x)
        return T_left * (1 - x) + T_right * x + D * N

Innovación Principal

La clave de esta extensión es que la red aprende a resolver la ecuación para cualquier condición de frontera dentro de un rango definido. Esto se logra alimentando a la red no solo con las coordenadas espaciales, sino también con los valores de frontera como entradas adicionales durante el entrenamiento.

📉 Evolución del Entrenamiento (5000 épocas)

El modelo fue entrenado durante 5000 épocas con condiciones de frontera aleatorias entre -20 y 50, logrando una pérdida final de 2.36e-3 y un mínimo de 9.48e-6. Las oscilaciones se deben a la aleatoriedad de las condiciones de frontera en cada batch.

📊 Resultados y Análisis

Los resultados del entrenamiento muestran:

  • Error relativo promedio: < 1% para la mayoría de configuraciones dentro del rango de entrenamiento [-20, 50]
  • Tiempo de inferencia: ~0.01 segundos por predicción (vs minutos para métodos numéricos tradicionales)
  • Pérdida final: 2.36e-3 después de 5000 épocas
  • Mejor pérdida: 9.48e-6 durante el entrenamiento

Observaciones importantes:

  • El error es prácticamente nulo en las fronteras (x=0 y x=1) gracias a la imposición exacta de condiciones
  • El mayor error se concentra en la región intermedia del dominio
  • Las oscilaciones en el entrenamiento se deben a la aleatoriedad de las condiciones de frontera en cada batch
  • El modelo generaliza bien para condiciones dentro del rango de entrenamiento [-20, 50]
